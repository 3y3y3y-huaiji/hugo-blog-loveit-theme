## 问题记录与解决方案

1. **项目重构与品牌统一**
   - 问题：网站原本属于个人博客（“安卓人”），部分文章带有个人信息，与全自动 AI 博客定位不符。
   - 解决方案：全面更改配置参数与关于页面，用 AI 绘图替换旧头像，并批量更新存量文章中的作者元数据。
   - 状态：已解决

2. **双平台 baseURL 渲染冲突**
   - 问题：GitHub Pages 运行在 `/hugo-blog-loveit-theme/` 子路径，Cloudflare Workers Assets 运行在根目录 `/`。静态配置会导致其中一处样式和资源链接失效。
   - 解决方案：在 `hugo.toml` 中保留 GitHub Pages 的 baseURL 配置，通过 npm 构建脚本 `build:cloudflare` 使用命令行参数 `hugo --baseURL="..."` 动态重写 Cloudflare 构建的基准路径。
   - 状态：已解决

3. **GitHub Actions 的 Node 运行期警告**
   - 问题：CI 中旧版的 actions（如 v3/v4 等）使用已废弃的 Node.js 20 运行时，提示建议升级。
   - 解决方案：升级所有 Actions 插件（如 `checkout@v6`, `setup-node@v6`, `upload-pages-artifact@v5`, `deploy-pages@v5`）至最新版本，全面切入 Node 24 执行环境。
   - 状态：已解决
