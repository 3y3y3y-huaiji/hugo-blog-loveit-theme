# 目录结构规范

## 1. 项目根目录结构

```
hugo-blog-loveit-theme/
├── .github/              # GitHub 相关配置（如 Actions 工作流）
├── archetypes/           # Hugo 内容原型模板
├── assets/               # 可被 Hugo 处理的资源文件
├── content/              # 博客内容（Markdown 文件）
├── data/                 # 数据文件（如 JSON、YAML）
├── docs/                 # 项目文档
├── i18n/                 # 国际化翻译文件
├── layouts/              # 自定义布局模板（覆盖主题）
├── public/               # 构建输出目录（Git 忽略）
├── resources/            # Hugo 资源缓存目录（Git 忽略）
├── static/               # 静态资源文件（直接复制到 public）
├── themes/               # Hugo 主题目录
├── .gitignore            # Git 忽略配置
├── hugo.toml             # Hugo 主配置文件
├── README.md             # 项目说明文档（中文）
└── README.en.md          # 项目说明文档（英文）
```

## 2. 主要目录功能定位

### 2.1 .github/
- 包含 GitHub Actions 工作流配置
- 用于自动化部署、测试等

### 2.2 archetypes/
- 包含 Hugo 内容原型模板
- 定义新文章的默认 Front Matter 和内容结构

### 2.3 assets/
- 包含可被 Hugo 处理的资源文件（如 SCSS、JavaScript）
- 支持资源管道处理（压缩、合并等）

### 2.4 content/
- 包含博客所有内容（Markdown 文件）
- 按内容类型和结构组织
  - `content/about/` - 关于页面
  - `content/posts/` - 博客文章
  - `content/tags/` - 标签页面（自动生成）
  - `content/categories/` - 分类页面（自动生成）

### 2.5 data/
- 包含数据文件（JSON、YAML、TOML）
- 可在模板中使用这些数据

### 2.6 docs/
- 包含项目文档
  - `docs/需求文档.md` - 项目需求文档
  - `docs/待办清单.md` - 项目待办任务
  - `docs/项目状态.md` - 项目进度状态
  - `docs/目录结构规范.md` - 本规范文件
  - `docs/文件命名规范.md` - 文件命名规范
  - `docs/代码风格规范.md` - 代码风格规范

### 2.7 i18n/
- 包含国际化翻译文件
- 支持多语言网站

### 2.8 layouts/
- 包含自定义布局模板
- 用于覆盖主题的默认模板
- 结构与主题模板保持一致

### 2.9 public/
- 包含 Hugo 构建后的输出文件
- 用于部署到服务器
- 已在 .gitignore 中配置忽略

### 2.10 resources/
- 包含 Hugo 资源缓存文件
- 已在 .gitignore 中配置忽略

### 2.11 static/
- 包含静态资源文件（直接复制到 public）
- 如图片、CSS、JavaScript 等
- 结构：
  - `static/css/` - 自定义 CSS 文件
  - `static/images/` - 图片资源
  - `static/js/` - 自定义 JavaScript 文件
  - `static/lib/` - 第三方库文件

### 2.12 themes/
- 包含 Hugo 主题
- 使用 Git 子模块管理

## 3. 目录结构管理原则

1. **遵循 Hugo 标准结构**：保持与 Hugo 推荐的目录结构一致
2. **功能分离**：不同类型的文件放在不同的目录
3. **层次清晰**：避免过深的目录层级（建议不超过 3 层）
4. **命名规范**：目录名使用小写字母，单词间用短横线分隔
5. **避免冗余**：删除不使用的空目录
6. **保持整洁**：定期清理临时文件和缓存目录
