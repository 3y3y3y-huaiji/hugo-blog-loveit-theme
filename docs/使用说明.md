# 使用说明文档

## 1. 项目概述

这是一个使用Hugo和LoveIt主题构建的个人博客项目，部署在GitHub Pages上。

## 2. 本地开发环境

### 2.1 安装Hugo

确保本地已安装Hugo扩展版本（Hugo Extended）。可以通过以下命令检查Hugo版本：

```bash
hugo version
```

### 2.2 克隆仓库

```bash
git clone https://gitee.com/su-mingkai1/hugo-blog-loveit-theme.git
git clone https://github.com/3y3y3y-huaiji/hugo-blog-loveit-theme.git
```

### 2.3 启动本地服务器

```bash
# 进入项目目录
cd hugo-blog-loveit-theme

# 启动本地开发服务器
hugo server

# 带草稿预览的开发服务器
hugo server -D
```

访问 `http://localhost:1313/hugo-blog-loveit-theme/` 查看本地预览。

## 3. 内容管理

### 3.1 创建新文章

使用Hugo的`new`命令创建新文章：

```bash
hugo new posts/文章标题.md
```

新文章将在 `content/posts/` 目录下创建，使用Markdown格式。

### 3.2 文章格式

文章的Front Matter（前置元数据）示例：

```yaml
---
title: "文章标题"
date: 2023-01-01T10:00:00+08:00
draft: false
tags: ["技术", "Hugo"]
categories: ["博客"]
---

这里是文章内容，支持Markdown格式。
```

### 3.3 图片管理

将图片放置在 `static/images/` 目录下，在文章中引用：

```markdown
![图片描述](/images/图片文件名.jpg)
```

## 4. 主题配置

主题配置主要在 `hugo.toml` 文件中进行：

### 4.1 基本配置

```toml
baseURL = 'https://3y3y3y-huaiji.github.io/hugo-blog-loveit-theme/'
languageCode = 'zh-cn'
title = '我的Hugo博客'
theme = 'LoveIt'
```

### 4.2 菜单配置

```toml
[menu]
  [[menu.main]]
    identifier = 'home'
    name = '首页'
    url = '/'
    weight = 1
  [[menu.main]]
    identifier = 'posts'
    name = '文章'
    url = '/posts/'
    weight = 2
  [[menu.main]]
    identifier = 'about'
    name = '关于'
    url = '/about/'
    weight = 3
```

### 4.3 其他配置

详细配置请参考LoveIt主题文档。

## 5. 部署流程

### 5.1 GitHub Pages自动部署

项目已配置GitHub Actions自动部署流程，当推送到GitHub的`main`分支时，会自动构建并部署到GitHub Pages。

### 5.2 手动部署（可选）

```bash
# 构建站点
hugo --gc --minify --cleanDestinationDir

# 部署到GitHub Pages
# 注意：需要先将public目录初始化为git仓库
cd public
git add .
git commit -m "更新站点内容"
git push origin main
```

## 6. 维护指南

### 6.1 定期更新

- 更新Hugo版本：`choco upgrade hugo-extended`（Windows）或 `brew upgrade hugo`（macOS）
- 更新主题：`cd themes/LoveIt && git pull`

### 6.2 常见问题

1. **本地预览异常**：
   - 检查Hugo版本是否与主题兼容
   - 检查配置文件语法是否正确

2. **构建失败**：
   - 检查文章格式是否正确
   - 检查图片引用路径是否正确

3. **部署失败**：
   - 检查GitHub Actions日志
   - 检查baseURL配置是否正确

## 7. 代码规范

- 文件命名：使用小写字母和连字符，避免中文和特殊字符
- 代码风格：遵循项目统一的代码风格规范
- 提交信息：使用清晰的提交信息，遵循Conventional Commits规范
